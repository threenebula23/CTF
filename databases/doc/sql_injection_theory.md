# Теория SQL-инъекций

## Что такое SQL-инъекция?
SQL-инъекция (SQLi) — это тип уязвимости безопасности веб-приложений, при которой злоумышленник может вмешиваться в запросы, которые приложение делает к своей базе данных. Это позволяет просматривать данные, которые обычно не предназначены для просмотра, изменять или удалять их.

## Основные типы SQL-инъекций

### 1. In-band SQLi (Классическая)
Самый распространенный тип, когда злоумышленник использует один и тот же канал связи для атаки и сбора результатов.

- **Error-based SQLi**: Использование сообщений об ошибках БД для получения информации о структуре.
- **Union-based SQLi**: Использование оператора `UNION` для объединения результатов оригинального запроса с результатами вредоносного.

### 2. Inferential SQLi (Слепая / Blind)
Злоумышленник не видит данных напрямую, но может делать выводы, задавая базе данных вопросы \"да/нет\".
- **Boolean-based**: Ответ страницы меняется в зависимости от того, истинно условие или ложно.
- **Time-based**: База данных делает паузу перед ответом, если условие истинно.

### 3. Out-of-band SQLi
Редкий тип, когда злоумышленник заставляет БД отправить данные по другому протоколу (например, HTTP или DNS).

## Примеры полезных нагрузок (Payloads)
- `' OR '1'='1` — обход авторизации.
 `' UNION SELECT username, password FROM users--` — кража учетных данных.
 - `; DROP TABLE users--` — удаление данных.
 
 ## Как защититься?
 1. **Использование параметризованных запросов (Prepared Statements)** — самый эффективный метод.
 2. **Валидация входных данных** (белые списки).
 3. **Принцип наименьших привилегий** для пользователя базы данных.
 4. **Экранирование спецсимволов** (менее надежно, чем параметры).
